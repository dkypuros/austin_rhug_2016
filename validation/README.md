# validation/

Independent reviews of the demo and presentation. Each review is a dated
markdown file capturing what was checked, by which reviewer/agent, the
findings, and what action was taken.

These reports are read-only artifacts. They are not a runbook; they are
evidence that the work was reviewed before being shown publicly.

## Index

| Date | Report | Subject |
|---|---|---|
| 2026-04-30 | [`2026-04-30-presentation-review.md`](./2026-04-30-presentation-review.md) | Pre-talk review of the public deck, voice tracks, and shipped RAG path |

## Conventions

- One report per review session.
- Filename: `YYYY-MM-DD-<scope>.md`.
- Each report ends with a "ship list" — the concrete actions that came out of it.
- If a review surfaces something that contradicts an earlier review, link both files and resolve in the newer report.
