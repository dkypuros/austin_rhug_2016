The big picture

Two AI harnesses author the same project. Their work becomes one Argo CD-managed app on Red Hat OpenShift AI. NVAIE entitles us to the NIMs; NGC is the catalog and hosted endpoints where we exercise them. The app speaks the same OpenAI contract whether it's calling NGC or the in-cluster vLLM on RHOAI. Microservices and inference, one platform, fluid swap.

What we built

Test in the cloud. Commit on-prem. Same contract end-to-end.

One client · two backends · one commit between them.

  • Use AI harnesses to explore on NGC (the NVAIE test surface) — models, retrieval, evaluation — without committing infrastructure.
  • Same OpenAI-compatible client. Three env vars choose the backend.
  • When the use case fits, promote it on-prem on RHOAI through Argo CD + Tekton — same Route, same UI, no code change.
  • Microservices and inference stay interchangeable until you're ready to commit. Issue #2 extends the same seam to embeddings + reranking for full RAG.
